Lemon Programming.

October 11, 2015 • [lemon] • [programming] • Anubhav Saini



So you want to program like a lemon.

You first need to learn what lemon programming is about.

Check out the list below.

Basic Constructs


  • You need to make sure that switch works. Really works.

      switch(number){  
  case 0:  
      if(number != 0){ abort(); }  
      else /* do something */  
  }


  • You need to mix spaces with tabs.

      void main () {  
      int i = 0;  
      if( !i ){  
        /* do this */  
      }  
      else  
          /* do that */  
    return 0;  
  }


  • You need to mix {} with (lack of {})

      if( number == 0){
      number = 1;
  }
  else
      number = 0;


  • You need to save bytes where possible. So use = instead of == and == instead of ===

      if( number = 0){
      number = 1;
  }
  else
      number = 0;


  • You need to implement cache first and when it hurts, refactor it.


  • If it still hurts, remove the cache implemented sofar and start from scratch.


  • You need to grow a pair of oo and tell yout team that you will test your code at run time. Who needs pesky test cases anyway: only lame programmers. Not you!


  • You need to refactor the code every other day. It doesn’t matter if it is already optimized. Make it better. Make it better.


Lemon Programming Naming Conventions

  • Use emotion revealing names


  • If code is lucidly telling it’s function and reason of existence upon first reading then use ambiguous variable names.


  • Use Alpha-Numeric names.


  • To make variable names searchable use hungarian notation.


  • Prefix names with class-name-identifier or short-class-name; which can be obtained by removing vowels most of the time.


  • Prefix Interfaces with I, Classes with K, Functions with F, Variables with V, Constants with C.

      interface IRobot {
      FTalk();
  }
  class KRobot : IRobot {
      const CID {public get; set;}
      void FTalk(){
          /* do something */
      }
  }

See how clear IRobot, KRobot, CID and FTalk are; one look and you know what each of them stands for.


  • Keep variable names as short as possible.


  • Be creative with the names.
